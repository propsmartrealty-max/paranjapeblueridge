# Paranjape Blue Ridge — Sovereign SEO Architecture Guide

This document details the technical SEO architecture, structured schema graphs, multilingual mapping, and build safeguards implemented for the Paranjape Blue Ridge portal.

---

## 🚀 1. Programmatic SEO (PSEO) Mesh
To outrank generic search aggregators, the site generates a granular web of **2,050+ targeted landing pages**.
* **Generator Source**: [src/data/seo-matrix.ts](file:///Users/vikasyewle/blueridge/src/data/seo-matrix.ts) contains themed keyword matrix arrays (corporate proximity, infrastructure projects, competitor comparisons, NRI investment yields).
* **Route Mapping**: The `generatePseoUrls()` function dynamically pairs configuration metrics (e.g., flat sizes) with local landmarks, building intent-focused slugs.
* **Component Rendering**: The catch-all dynamic page [src/app/[slug]/page.tsx](file:///Users/vikasyewle/blueridge/src/app/[slug]/page.tsx) resolves these slugs. It delegates rendering to the SSR component [src/components/PseoLandingPage.tsx](file:///Users/vikasyewle/blueridge/src/components/PseoLandingPage.tsx), which pulls optimized content blocks from [src/components/DynamicContentBlock.tsx](file:///Users/vikasyewle/blueridge/src/components/DynamicContentBlock.tsx).

## 🌐 2. Multilingual Alternate Mapping (hreflang)
To capture vernacular search intent without diluting page authority:
* **Translation Prefix**: Regional Marathi pages use the `mr-` prefix (e.g., `mr-2-bhk-flats-in-hinjewadi-phase-1`).
* **Sitemap Alignment**: Inside [src/app/sitemap.ts](file:///Users/vikasyewle/blueridge/src/app/sitemap.ts), alternate pairs are mapped via Next.js sitemap configurations:
  ```typescript
  alternates: {
    languages: {
      'en': EnglishUrl,
      'mr': MarathiUrl,
    }
  }
  ```
* **Page-Level Injection**: The PSEO metadata engine inside [src/app/[slug]/page.tsx](file:///Users/vikasyewle/blueridge/src/app/[slug]/page.tsx) automatically injects matching `en-IN` and `mr-IN` alternates in the HTML header.

## 📊 3. Consolidated Structured Schemas (JSON-LD)
All dynamic schemas are managed inside [src/components/JSONLD.tsx](file:///Users/vikasyewle/blueridge/src/components/JSONLD.tsx) as a single Graph array:
* **Organization & WebSite**: Connects the developer (Paranjape Schemes) with Wikipedia authority bridges (`sameAs`) and defines default search capabilities.
* **ApartmentComplex & Place**: Sets geotags, price specifications, RERA certificates, and 138-acre township amenities (golf course, school, boat club).
* **LocalBusiness with Reviews**: Includes rating averages mapped across diversified resident reviews to verify organic trust score credibility.
* **RealEstateListing & Product**: Declares precise units, pricing bounds, and layouts.
* **FAQPage & HowTo**: Supplies dynamic, rich snippet Q&As targeting connection commutes, Metro Line 3, and purchase guidelines.

## 🛡️ 4. Build-Time Schema Validation Safeguard
To prevent data regressions or broken routes from deploying:
* **Runner Script**: [scripts/verify-schemas.ts](file:///Users/vikasyewle/blueridge/scripts/verify-schemas.ts) runs automatically prior to Next.js compilation.
* **Checks Executed**:
  1. **Route Duplication**: Guarantees zero overlapping slugs across PSEO silos.
  2. **URL Semantics**: Asserts that all slugs conform to lowercase canonical formats.
  3. **RERA Registry Audit**: Confirms all projects specify registration numbers.
  4. **Date Integrity**: Asserts news article dates parse cleanly.
  5. **Absolute Path Verification**: Asserts all schema-rendered assets use secure (`https://`) absolute domains.
* **Build Lock**: If any check fails, the pipeline halts immediately. Configured in [package.json](file:///Users/vikasyewle/blueridge/package.json#L7).

## ⚡ 5. Caching & Edge Performance
* **Dynamic OG Cards**: Custom card images generated by `/api/og` utilize long-term cache headers (`Cache-Control: public, max-age=31536000, immutable`). These are cached instantly at Vercel CDN layers, reducing sharing latency to 0.0s.
* **Preconnected DNS Links**: Crucial external resources (Google Tag Manager, Google Analytics, developer domain) are pre-connected inside the HTML layout to decrease TCP handshake times.
